# ブロックエディタ 技術スタック & 設計仕様（Claude Code 引き継ぎ書 / gpui ネイティブ版）

> 思考を深めるための、シンプル・キーボードファースト・AI統合のブロックエディタ。
> Zed 製ライブラリ（gpui）を使い、Rust ネイティブのフルスタックで構築する。

---

## 1. このプロダクトは何か（一行定義）

**フォルダ内の `.md` ファイル群を、キーボードファーストで編集する、シンプルでAI統合されたネイティブエディタ。**
Obsidian 的な知識ベース（md ファイル群＋リンク）の思想を、Zed と同じ gpui 基盤の上に、独自のシンプルな書き味で実装する。

---

## 2. コア要件（確定）

| # | 要件 | 意味 |
|---|------|------|
| 1 | **Markdown で保管** | 保存フォーマットが `.md`。独自バイナリ/DBに閉じ込めない。他ツールで開ける・Git差分が見える・ロックインなし |
| 2 | **ブロック追加は `/` コマンド** | 新規ブロックはハンドルやボタンでなく `/` から呼ぶ。マウス不要 |
| 3 | **編集もキーボードファースト** | 既存ブロックの編集・移動・変換も同じ思想。追加と編集のUIの一貫性 |
| 4 | **ナレッジ管理（A/B/C 全部）** | A:リンク / B:階層（フォルダ）/ C:日付流し（デイリーノート）。土台は「フォルダ内の `.md` 群」一つで全部成立 |

### 「シンプルにしたい」対象（最優先の価値観）
- **UI** — 画面に余計なものを出さない。IA Writer 的な静けさ。ハンドル/ホバーメニュー/ツールバーを出さない
- **キーボードファースト** — 手をホームポジションから離さない。`/` が司令塔。マウスは使えるが主役でない。Vim のモード制は不採用（モードレスで速い＝Zed 路線）
- **AI統合** — AI が後付け機能でなく書く体験に溶け込む。ただし盛りすぎない

---

## 3. 確定した設計判断

- **背骨は線形ブロック** — 縦一列が基本。座標管理（自由配置キャンバス）は持たない
- **ブロックは「裏方の構造」** — 普段はただのテキストに見え、ブロックUIを画面に出さない。`#` `-` 等の Markdown を打つと構造化される（"打って化ける"）
- **`/` が操作の入口** — ブロック追加・種類変換・移動などを `/` から呼ぶ。覚えるのは最小限
- **表現は豊かに、到達は一瞬に** — URL/テーブル/画像もフルサポート。到達手段は「打って化ける」＋「`/` で呼ぶ」。表現は削らず操作だけ削る
- **空間は妥協** — 専用キャンバスは作らない。リンクから自動生成される **グラフビュー**＋**フォーカスモード**（一点を明るく他を薄く）で「関係を見る/集中して潜る」欲求に応える
- **保存はサイレント** — 「保存ボタン」概念なし。自動保存

### 未決（実装しながら詰める）
- **AI統合の絡ませ方** — 第一候補は **「選んで頼む」**（テキスト選択→ショートカット/`/` で要約・続き生成。自分が主導、AIは勝手に出ない＝最もシンプル）。他案：カーソル直下のAI指示エリア／裏で先回り（Copilot型、シンプルさと緊張）
- **`/` の範囲** — 追加専用か、移動/変換/削除まで全部 `/` から呼ぶ万能入口にするか
- **補完の「呼吸」** — 候補をいつ出すか。シンプルさ優先なら「普段は黙る、呼ばれたら出す」に倒れる。実機で要調整

## 4. 技術スタック（gpui ネイティブ・フルスタック）

| 領域 | 採用 | 備考 |
|------|------|------|
| 言語 | **Rust** | |
| GUIフレームワーク | **gpui**（Zed 製） | GPU レンダリングの高速 UI 基盤。Zed と同じ路線で「軽量・無遅延」を狙う |
| UIコンポーネント | **gpui-component**（longbridge 製、0.5.x） | 60+ コンポーネント。Markdown/簡易HTMLレンダリング・Dockレイアウト・仮想化List/Table を内蔵 |
| エディタ土台 | gpui-component の **Editor / InputState（multi_line）** | 最大20万行対応・LSP・シンタックスハイライト。Tree-sitter + Rope。`editor` example を出発点に |
| テキスト構造 | **Rope**（gpui-component が採用済み） | 巨大文書でも軽い差分編集 |
| シンタックス | **Tree-sitter**（同梱） | Markdown のパース/ハイライトに流用 |
| ドキュメントモデル | **自作** | ブロックツリー（線形＋畳み）。保存時に `.md` へシリアライズ |
| 永続化 | `.md` ファイル入出力 | 保存ボタンなし・自動保存。フォルダ＝ナレッジベース |
| リンク/バックリンク | 自作（索引） | `[[file]]` を解析し逆引き索引を裏スレッドで構築 |
| グラフビュー | 自作 | リンク関係を可視化（自由配置でなく自動レイアウト） |
| AI | LLM API を非同期呼び出し | 裏スレッド。主スレッドを待たせない |

### 重要な実装メモ（最新状況）
- **gpui / gpui-component は git main 版を使う**。crates.io 公開版より進んでおり、本体に未公開の新機能がある。`Cargo.toml` で git 依存を指定するのが公式推奨
- gpui は pre-1.0 で破壊的変更が頻繁。**UI層を薄く保ち、gpui 依存を局所化**しておく（差し替えやすさの保険）
- 出発点として、まず `cargo run --example editor`（LSP対応エディタ）と `cargo run --example markdown` を動かして構造を把握すると速い

## 5. アーキテクチャ（3層・主スレッドは待たせない）

```
UI層（gpui / gpui-component）  : 書く地点／`/`コマンドパレット／AI呼び出しUI  ← 主スレッド・無遅延を死守
コア層（自作）                 : ブロックツリー／編集オペ／Markdownシリアライズ／補完エンジン
サービス層（裏スレッド）        : .md入出力／AI呼び出し／リンク索引・バックリンク・グラフ構築  ← 遅くてよい
```
**境界線：主スレッドは絶対に待たない。** AIも保存も索引構築も裏に逃がし、結果は非同期で主へ返す。
速い補完（コア層・同期・ルールベース）と賢いAI（サービス層・非同期）をスレッドで物理的に住み分ける。

---

## 6. 補完エンジンの設計

肝：**「候補を作る人」と「いつ出すか決める人」を分離する。** 呼吸の調整は後者だけ差し替える。

```
入力イベント → コンテキスト抽出 → 候補プロバイダ群（複数の小さな専門家）
   → ゲート（集約・順位付け・"今は黙る"判断）
   → ポリシー（呼吸：いつ/何個/どれだけ待つ ← ここだけ差し替えて調整）
   → 提示（カーソル直下に候補）→ 受領(Tab)/棄却(Esc)
   → 学習ログ（採否を記録しポリシーに反映。最初は入れ物だけ）
```
- プロバイダは複数並べる（`#`→見出し、リスト途中→次項目、URL検知→リンクカード…）。後でAIプロバイダを1個足せる構造に
- ルールベースなら全部同期・無遅延。AIプロバイダは非同期で後から候補を差し込む
- シンプルさ優先の初期方針：**普段は黙る。`/` や `#` で呼ばれた時だけ出す**

---

## 7. Claude Code への最初の依頼例 & マイルストーン

最初の依頼例：
> 「gpui-component を git 依存で取り込み、`editor` example をベースに、ローカルの `.md` ファイルを1枚読み込んで表示・編集・自動保存できる最小アプリを作って。ウィンドウは単一ペインでよい。」

マイルストーン：
1. gpui-component の `editor` / `markdown` example を `cargo run` で動かす（足場確認）
2. `.md` を1枚開いて表示できる
3. キーボードで編集し、変更が自動で `.md` に保存される
4. `/` を押すとコマンドパレットが出て、ブロック（見出し/リスト/コード/テーブル/画像/リンク）を挿入できる
5. `#` `-` などの Markdown を打つと即座に構造化される（"打って化ける"）
6. テキスト選択 → ショートカットで AI（要約/続き生成）を呼べる ← AI統合の第一歩
7. `[[file]]` リンクを解析し、バックリンク一覧を表示
8. リンクからグラフビューを生成

## 付記：このプロジェクトの背骨（議論の結論）

最初は「キャンバス型に惹かれるが操作感が死ぬ」という矛盾から出発した。
辿り着いた答えは、**自由配置の空間を諦め、線形ブロック＋リンク（グラフビュー）＋フォーカスモードで"空間の旨味"だけを取る**こと。
全要件は **「フォルダ内の `.md` ファイル群を、キーボードファーストで編集する」** の一文に集約される。
シンプルさ（UI・キーボードファースト・AI統合）が、全設計判断のモノサシ。
土台は Zed と同じ gpui に乗せ、車輪の再発明（テキストレンダリング・IME・無遅延描画）を避けて、書き味と AI統合という"自分の理想"の部分に集中する。
